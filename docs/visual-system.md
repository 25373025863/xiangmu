# GAME // SELECT 视觉规范

更新时间：2026-07-15。活动前端为 `frontend/`，所有新页面和组件应遵守本规范。

## 设计方向

- 风格：赛博朋克游戏工作台，不使用霓虹紫色渐变或装饰性光球。
- 媒体：首屏和精选内容优先展示真实游戏画面；图片必须包含 `alt`，非关键图片使用 `loading="lazy"`。
- 信息层级：品牌、当前任务和主要操作优先；非关键内容在首屏之后呈现。
- 交互：青蓝表示可操作状态，洋红表示主要行动，荧黄表示服务正常或确认成功，红色表示风险或删除。

## 色彩令牌

| 用途 | 令牌 | 值 |
| --- | --- | --- |
| 页面背景 | `--bg` | `#07090f` |
| 表面 | `--surface` | `#101827` |
| 主文字 | `--ink` | `#e8f6ff` |
| 次要文字 | `--muted` | `#aab7cf` |
| 可操作色 | `--primary` | `#31dff5` |
| 主要行动 | `--accent` | `#ff3cad` |
| 成功信号 | `--signal` | `#cafc4a` |
| 风险操作 | `--danger` | `#ff6978` |

组件中不得新增孤立的原始颜色；优先复用以上令牌。

## 排版与空间

- 字体：`Bahnschrift`、`Segoe UI`、`Microsoft YaHei` 回退链。
- 正文最小 `14px`，移动端输入和正文保持至少 `16px` 的可读尺寸。
- 间距采用 8px 节奏；大区块使用 56px、60px 或 88px 的垂直间距。
- 页面内容宽度为 `min(1180px, 100% - 40px)`；移动端改为 28px 总边距。
- 圆角上限为 6px；阴影只用于层级和焦点，不做装饰性悬浮卡片。

## 组件与状态

- 所有按钮最低高度 44px，主要 CTA 每个视图只保留一个。
- 图标使用 Lucide，图标按钮必须有 `aria-label` 或 `title`。
- 焦点使用 3px 荧黄轮廓；颜色不能作为唯一的状态说明。
- 过渡限定在 180-220ms；`prefers-reduced-motion` 下关闭动画。
- 小屏导航允许换行，不允许主内容区域产生横向滚动。

## 首页结构

1. 全宽真实游戏画面首屏：品牌、应用简介、开始推荐和游戏库入口。
2. 三步工作流：偏好、推荐、保存。
3. 可检索游戏库精选。
4. API Key 安全提示与设置入口。

首屏需在桌面和移动视口中露出下一段内容的一部分，避免把后续信息完全藏在首屏下方。
